# Kashidasu実装規約

## 変数・定数名

変数は`let`, `const`で宣言し、`camelCase`を使用してください。

## 関数名

関数は`camelCase`を使用してください。

## コントローラー・モデル・ルーターのファイル命名基準

- コントローラーは`bookController.js`のように`Controller`を末尾に付与してください。
- モデルは`bookModel.js`のように`Model`を末尾に付与してください。
- ルーターは`bookRoutes.js`のように`Routes`を末尾に付与してください。

## モデル層の処理規約

- モデル層の関数は`Promise`を返すようにしてください。
- 成功時は`{ success: true, data: ... }`の形式で返す
- 失敗時は`{ success: false, message: "エラーメッセージ" }`の形式で返す
- モデル層の関数は`try-catch`で囲み、エラーが発生した場合は`throw`してください。
- `throw`した場合は、トランザクションをロールバックしてください。

## jsonキー

jsonのキーは`camelCase`を使用してください。

## APIエンドポイント規約

- APIエンドポイントは`/api/book`のように`/api/`で始まり、単数形の名前を使用してください。
- `-`や`_`を使用せず、`/`で区切ってください。
- 複数のリソースを取得する場合は、`/api/book/all`のように`/all`を末尾に付与してください。
- 特定のリソースを取得する場合は、`/api/book/one/:id`のように`/one`を末尾の1つ前に付与してください。
- APIメソッドは`GET`, `POST`, `PATCH`, `DELETE`を使用してください。
- リクエストボディは`JSON`を使用してください。
- レスポンスボディは`JSON`を使用してください。
- レスポンスには必ず`success(ブール型)`と`message(文字列型)`を含めてください。
- エラーレスポンスは`{ "success": false, "message": "message" }`の形式で返してください。

## コメントアウト

- 基本的にコメントアウトによる説明は、以下の場合のみ行い、それ以外は行わない。
  - 処理にマジックナンバーを含む場合、その意味をコメントアウトで記述する。
- 関数を定義する際、その関数の目的や処理内容を下記のように記述する。

```javascript
/**
 * 関数の目的や処理内容
 * @param {paramType} paramName - パラメータの説明
 * @returns {returnType} 戻り値の説明
 */
```

- ファイルの先頭には、ファイルの目的や処理内容を下記のように記述する。

```javascript
/**
 * ファイルの目的や処理内容
 */
```

## コンソールログ

エラーログ以外のコンソールログは、デバッグ以外で使用せず、削除してください。

ただし、以下の場合は許可されます。

- サーバー起動時のログ出力
- その他必要な場合
